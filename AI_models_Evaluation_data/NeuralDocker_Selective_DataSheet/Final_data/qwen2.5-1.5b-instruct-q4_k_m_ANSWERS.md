**Q222.** To check if a binary tree is balanced, we can use the concept of height-balanced trees. A binary tree is considered balanced if for every node, the heights of its left and right subtrees differ by no more than one.

Here's how you could implement this in Python:

```python
class TreeNode:
    def __init__(self, value=0):
        self.value = value
        self.left = None
        self.right = None

def is_balanced(root):
    """
    Check if the binary tree rooted at 'root' is balanced.
    
    :param root: TreeNode, the root of the binary tree
    :return: bool, True if the tree is balanced, False otherwise
    """
    def check_height(node):
        if node is None:
            return 0
        
        # Recursively get the height of each subtree
        left_height = check_height(node.left)
        right_height = check_height(node.right)
        
        # If either subtree is not balanced, return -1 to indicate imbalance
        if left_height == -1 or right_height == -1:
            return -1
        
        # Check if the current node's height exceeds 1 from its children
        if abs(left_height - right_height) > 1:
            return -1
        
        # Return the height of this subtree, plus one for the current node
        return max(left_height, right_height) + 1
    
    # Start checking from the root
    return check_height(root) != -1

# Example usage:
if __name__ == "__main__":
    # Creating a balanced binary tree
    root = TreeNode(1)
    root.left = TreeNode(2)
    root.right = TreeNode(3)
    root.left.left = TreeNode(4)
    root.left.right = TreeNode(5)
    
    print(is_balanced(root))  # Output: True
    
    # Creating an unbalanced binary tree
    root = TreeNode(1)
    root.left = TreeNode(2)
    root.right = TreeNode(3)
    root.left.right = TreeNode(4)

This function `is_balanced` uses a helper function `check_height` to recursively calculate the height of each subtree and check if it is balanced. If at any point the difference in heights between left and right subtrees exceeds one, the tree is unbalanced.

```python
def binary_search(arr, target):
    """
    Perform binary search on a sorted array to find the index of 'target'.
    
    Parameters:

            return mid

        elif arr[mid] < target:

    return -1

4. If the target is greater than the middle element, it narrows down the search to the right half by moving `left` up.
5. If the target is less than the middle element, it narrows down the search to the left half by moving `right` down.

**Q226.** To find the longest common subsequence (LCS) of two strings, you can use dynamic programming. Here's a Python function that implements this approach:

```python
def lcs(s1, s2):
    m = len(s1)
    n = len(s2)

    # Create a 2D array to store lengths of LCS for substrings.
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    # Build the dp table
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if s1[i - 1] == s2[j - 1]:
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])

    # The length of the LCS is in the bottom-right cell.
    return dp[m][n]

# Example usage:

s2 = "AEDFHR"
print("Length of LCS is", lcs(s1, s2))
```

This function works as follows:

1. It initializes a 2D array `dp` where `dp[i][j]` will store the length of the longest common subsequence of substrings `s1[0..i-1]` and `s2[0..j-1]`.

2. It iterates through each character in both strings, comparing them:
   - If characters match (`s1[i-1] == s2[j-1]`), the length is incremented by 1.
   - Otherwise, the maximum value from the previous row or column (i.e., `dp[i-1][j]` or `dp[i][j-1]`) is taken.

3. The final answer is stored in `dp[m][n]`, which represents the length of the LCS for the entire strings `s1` and `s2`.

This approach has a time complexity of O(m * n) where m and n are the lengths of the input strings, making it efficient for large inputs.

                return False

            k += 1
            col -= 1

The 0/1 Knapsack problem is a classic optimization problem where you have a set of items, each with a weight and a value, and you want to determine the number of each item to include in a collection so that the total weight is less than or equal to a given limit and the total value is as large as possible. This can be solved using dynamic programming.

Here's how you could implement this:

```python
def knapsack(weights, values, capacity):
    """
    Solves the 0/1 Knapsack problem using Dynamic Programming.
    
    :param weights: List of weights for each item.
    :param values: List of values for each item.
    :param capacity: Maximum weight the knapsack can hold.
    :return: The maximum value that can be obtained with the given capacity.
    """
    n = len(weights)
    dp = [[0] * (capacity + 1) for _ in range(n + 1)]
    
    # Build the DP table
    for i in range(1, n + 1):
        for w in range(capacity + 1):
            if weights[i - 1] <= w:
                dp[i][w] = max(dp[i - 1][w], values[i - 1] + dp[i][w - weights[i - 1]])
            else:
                dp[i][w] = dp[i - 1][w]
    
    return dp[n][capacity]

# Example usage

values = [6, 7, 8, 9]
capacity = 5

print(knapsack(weights, values, capacity))  # Output should be the maximum value that can be obtained with the given capacity.
```

This example assumes that you have a dictionary representing the graph where keys are node identifiers and values are dictionaries containing adjacency lists.

```python
import heapq

def dijkstra(graph, start, end):
    # Initialize distances with infinity for all nodes except the starting node
    distances = {node: float('inf') for node in graph}
    distances[start] = 0
    
    # Priority queue to store (distance, node)
    priority_queue = [(0, start)]
    
    while priority_queue:
        current_distance, current_node = heapq.heappop(priority_queue)
        
        if current_distance > distances[current_node]:
            continue
        
        for neighbor, weight in graph[current_node].items():
            distance = current_distance + weight
            
            # Only consider this path if it's better than the previously recorded one
            if distance < distances[neighbor]:
                distances[neighbor] = distance
                heapq.heappush(priority_queue, (distance, neighbor))
                
    return distances[end]

    'B': {'A': 1, 'D': 2},
    'C': {'A': 4, 'D': 3},
    'D': {'B': 2, 'C': 3}
}

start_node = 'A'
end_node = 'D'

shortest_path_distance = dijkstra(graph, start_node, end_node)

```

**Q287.** The Observer design pattern is a behavioral design pattern that defines an object, called the subject, which maintains a list of its dependents, called observers. The subject notifies the observers about changes in its state by calling one of their methods.
Here's a simple Python implementation:

```python
class Subject:
    def __init__(self):
        self._observers = []

    def attach(self, observer):
        if observer not in self._observers:
            self._observers.append(observer)

    def detach(self, observer):
        try:
            self._observers.remove(observer)
        except ValueError:
            pass

    def notify(self):
        for observer in self._observers:
            observer.update()

class Observer:
    def update(self, subject):
        print(f"Observer received notification from {subject}")

# Example usage
if __name__ == "__main__":
    subject = Subject()
    
    class ConcreteSubject(Subject):
        pass
    
    class ConcreteObserver1(Observer):
        def update(self, subject):
            print("ConcreteObserver1 received notification from ConcreteSubject")
    
    class ConcreteObserver2(Observer):
        def update(self, subject):
            print("ConcreteObserver2 received notification from ConcreteSubject")
    
    observer1 = ConcreteObserver1()
    observer2 = ConcreteObserver2()
    
    subject.attach(observer1)
    subject.attach(observer2)
    
    subject.notify()  # Output: "Observer received notification from ConcreteSubject"
```

In this example, `Subject` is the main class that maintains a list of observers. The `attach`, `detach`, and `notify` methods allow adding, removing, and notifying observers respectively. The `Observer` class defines an update method that gets called when the subject notifies its observers.

def min_coins_for_change(amount, denominations):
    # Initialize a list to store the minimum number of coins needed for each amount from 0 to `amount`

def dfs(graph, start):
    visited = set()  # Set to keep track of visited nodes
    stack = [start]   # Initialize a stack with the starting node

**Q300.** Certainly! Below is a Python class that implements an LRU cache using the `collections` module. This example uses a dictionary to store key-value pairs and maintains a doubly linked list to keep track of the most recently used items.

```python
from collections import OrderedDict

class LRUCache:
    def __init__(self, capacity: int):
        self.capacity = capacity
        self.cache = OrderedDict()

    def get(self, key: str) -> int:
        if key not in self.cache:
            return -1  # Key not found
        
        value = self.cache.pop(key)
        self.cache[key] = value  # Move the accessed item to the end of the cache
        return value

    def put(self, key: str, value: int) -> None:
        if key in self.cache:
            self.cache.pop(key)  # Remove the old entry from the cache
        
        if len(self.cache) >= self.capacity:
            # Remove the least recently used item (first element of OrderedDict)
            oldest_key = next(iter(self.cache))
            del self.cache[oldest_key]
        
        self.cache[key] = value

# Example usage
cache = LRUCache(2)

cache.put('a', 1)
cache.put('b', 2)
print(cache.get('a'))  # Output: 1
cache.put('c', 3)      # Evicts 'b' since the cache is full.
print(cache.get('b'))  # Output: -1 (Key not found)
print(cache.get('c'))  # Output: 3
```
